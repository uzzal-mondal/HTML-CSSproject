Introduction The Hub Framework is built around the idea of a content-driven architecture, which just like content-driven UI design puts the content front and center - and lets the content drive the UI rather than the other way around.

The way this works is through the use of view models and component models that each encapsulates content on either the view level, or the component level. Components are the visual buildings blocks of a Hub Framework-powered view, and are used to render the content defined by a component model. (For more information about components, see the Component programming guide) Content hierarchy The content of a Hub Framework-powered view is described using a hierarchy of models, consisting of HUBViewModel, HUBComponentModel and HUBComponentImageData objects.
